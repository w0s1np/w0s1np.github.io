Tomcat Listener 内存马

什么是 Listener

Listener（监听器）就是一个实现特定接口的普通 java 程序，这个程序专门用于监听另一个 java 对象的方法调用或属性改变，当被监听对象发生上述事件后，监听器某个方法将立即被执行。

Tomcat Listener 流程分析

在 class 部分和requestInitialized()下断点，

根据堆栈回溯找到StandardContext.listenerStart()方法，

可以看到它先调用findApplicationListeners()获取 Listener 的名字，然后再实例化。然后遍历 result 中的 Listener，根据其不同类型放入不同的数组中。

这里的 ServletRequestListener 就放入 eventListeners 数组中。

然后通过调用getApplicationEventListeners()获取 applicationEventListenersList 属性(即已注册的 Listener)，但是在函数中首先会清空 applicationEventListenersList ，所以在后面又再重新取出来赋值 applicationEventListenersList，再后面就是往 applicationEventListenersList 中添加刚才获取到的 eventListeners 。

所以现在 applicationEventListenersList 数组只中存放的都是实例化后的 listener。

从上面可以知道 listenerStart() 将我们的 Listener 实例化添加到了 applicationEventListenersList 中，但是想触发 Listener 还需要一个函数点来调用才行，

就需要到第二个断点了，根据调用堆栈我们找到了fireRequestInitEvent()方法，

它调用了 getApplicationEventListeners() ，然后调用 listener.requestInitialized(event); 这里的 listener 就是上面注册过的 applicationEventListenersList 数组。

内存马注入流程分析

从上文分析可知，Listener 主要就是来自两个部分，一个就是 tomcat 初始化时从 web.xml 实例化的 Listener，一个就是 applicationEventListenersList 中的 Listener，我们能控制的就是第二个，所以我们就需要往 applicationEventListenersList 加入恶意 Listener。

在 StandardContext 中存在 addApplicationEventListener() 方法往 applicationEventListenersList 中加入 Listener。

  1. 继承并编写一个恶意 Listener
  2. 获取 StandardContext
  3. 调用StandardContext.addApplicationEventListener()添加恶意 Listener
